Funções em C

Blocos reutilizáveis de código.

Declaração:

  • tipo nomeFuncao(parâmetros) { ... }

Parâmetros:

  • Por valor: cópia da variável (não altera a original).
  • Por referência: com ponteiros (*ptr) para alterar diretamente.

Escopo de memória:

  • Variáveis locais: Stack.
  • Funções modificam dados com ponteiros.

TMatriz

Vetor de duas dimensões.

  • Acesso: matriz[i][j] para acessar um elemento.
  • Inicialização: int m[2][2] = {{1,2},{3,4}};
  • Alocação dinâmica: Usando int **matriz + malloc.
  • União: reúne todos os elementos de dois conjuntos.
  • Uso comum: tabelas, mapas, planilhas, jogos, gráficos.

Vetores em C

Estrutura linear que armazena dados do mesmo tipo.

Alocação de memória:

  • Variáveis locais (stack), globais (data/bss), dinâmicas (heap).

Tipos de vetores:

  • Estático: Tamanho fixo, acesso rápido.
  • Dinâmico: Usa malloc/realloc, tamanho variável.
  • Lista encadeada: Acesso lento, mas inserção/remoção eficiente.
  • Heap: Útil para estruturas com prioridade.

Ordenação:

  • Bubble Sort: simples, mas lento.
  • Selection Sort: busca o menor e coloca na frente.
  • QuickSort: rápido, usa recursão e divisão por pivô.

Busca:

  • Linear: percorre um por um.
  • Binária: só em vetores ordenados; mais rápida (O(log n)).

Structs

Agrupam dados de tipos diferentes em um único tipo.

Ponteiros:

  • Permitem passagem por referência e alocação dinâmica.
  • Acesso a campos via ponteiro: ptr->campo.

Uso prático:

  • Vetores de structs.
  • Alocação dinâmica com malloc.
  • Funções que modificam structs via ponteiro.

Nó

Um nó é uma unidade básica de uma estrutura de dados dinâmica como listas, árvores e grafos.

  • Ele armazena um valor (dado) e referência(s) para outro(s) nó(s).

Cada nó contém:

  • Um campo de dado (int valor).
  • Um ponteiro para o próximo nó (*prox).

O último nó aponta para NULL, indicando o fim da lista.

  • Cada nó aponta tanto para o próximo quanto para o anterior.
  • Permite navegação nos dois sentidos.

Nó em Árvores:

  • Usado em árvores binárias.
  • Cada nó aponta para dois filhos: esquerda e direita.

Listas Encadeadas

Cada nó tem um valor e um ponteiro para o próximo.

Operações:

  • Inserir no início/fim.
  • Remover por valor.
  • Percorrer e buscar elementos.

Vantagens:

  • Tamanho dinâmico.
  • Inserção/remoção eficiente.

Desvantagens:

  • Acesso sequencial (lento).
  • Consome mais memória (ponteiros).

Pilhas e Filas

Pilha (Stack):

  • LIFO (último a entrar, primeiro a sair).
  • Operações: push, pop, peek.
  • Exemplo: histórico do navegador.

Fila (Queue):

  • FIFO (primeiro a entrar, primeiro a sair).
  • Operações: enqueue, dequeue, peek.
  • Exemplo: fila de atendimento.

PPonteiro de ponteiro (**ptr): Necessário para modificar ponteiros em funções.

Fila com prioridade:

  • Elementos com mais prioridade são atendidos antes.

Elementos com mais prioridade são atendidos antes.

  • Cada nó tem ponteiros para anterior e próximo.
  • Arquiteturas RISC vs CISC
  • Permite navegação nos dois sentidos.

Fila com vetor:

  • Problema: desperdício de espaço após remoções.

Fila Circular:

  • Reutiliza posições com fim = (fim + 1) % TAM.

Lista Duplamente Encadeada Circular:

  • Primeiro nó aponta para o último e vice-versa.
  • Útil em simulações circulares (ex: jogos, playlists).
