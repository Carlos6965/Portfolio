Matéria ministrada por: Alexandre Aparecido Bernardes

DCL – Data Control Language

Conjunto de comandos SQL para administrar usuários, grupos e permissões no PostgreSQL.

Principais Comandos:

  • CREATE ROLE: Cria papéis (roles) com opções como LOGIN, SUPERUSER, PASSWORD, CONNECTION LIMIT, etc.
  • ALTER ROLE: Modifica papéis existentes (ex.: definir senha).
  • DROP ROLE: Remove papéis.
  • GRANT/REVOKE: Concede ou revoga privilégios (ex.: SELECT, INSERT) em objetos do banco.

Grupos: Papéis sem permissão de login, usados para agrupar usuários.

Exercícios Práticos: Incluem criação de papéis com restrições, concessão de permissões e testes de acesso.

Stored Procedures e Functions no PostgreSQL

Blocos de código SQL armazenados no servidor para melhorar desempenho, reutilização e segurança.

Diferenças:

  • Functions: Sempre retornam valores, podem ser usadas em consultas.
  • Procedures: Não retornam valores diretamente, executadas via CALL.

Tipos de Funções:

  • SQL: Comandos simples, sem estruturas de controle.
  • PL/pgSQL: Suporta variáveis, loops e condições.
  • Linguagens Externas: Como Python ou C.

Vantagens: Redução de tráfego de rede, centralização de regras de negócio, segurança.
